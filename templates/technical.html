Why XGBoost Won

Why XGBoost Excels

  • Captures non-linear relationships - Age and BMI don't affect cost linearly
  • Handles the "smoker effect" - Smokers pay 3-4x more, creating distinct groups
  • Detects feature interactions - Old + smoker = much higher than sum of parts
  • Built-in regularization - L1 & L2 prevent overfitting
  • Handles mixed data types - Works well with both categorical and numerical

Why Linear Models Struggled

  • Assumes linear relationships - Can only draw straight lines
  • Can't handle sharp jumps - Smoker vs non-smoker is a 3x jump, not gradual
  • No interaction detection - Treats each feature independently
  • Sensitive to outliers - High-cost smokers skew predictions
  • Limited expressiveness - y = w1*x1 + w2*x2 + ... is too simple

The Smoker Effect Explained

The biggest factor in insurance cost is smoking status. Here's why tree-based models handle this better:

# Linear Model thinks:
cost = 5000 + (300 Ã— age) + (10000 Ã— smoker) + ...

# Reality (what XGBoost captures):
if smoker == "yes":
    cost = 20000 + (500 Ã— age) + (800 Ã— bmi) # Different formula!
else:
    cost = 2000 + (250 Ã— age) + (300 Ã— bmi)

XGBoost essentially learns different pricing models for smokers vs non-smokers, while linear regression tries to use one formula for everyone.
